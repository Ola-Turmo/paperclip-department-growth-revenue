# Lessons Learned — Growth Revenue Department

This directory captures task-by-task learning artifacts, postmortems, and durable improvements.

## Purpose

Per the PRD's continuous learning loop:
- Every completed task must leave behind at least one durable improvement
- Capture problem, evidence, decision, outcome, and follow-up questions
- Track confidence and unknowns as research backlog items

## Learning Loop Requirements

1. **Before**: Read repo README, PRD, touched modules, tests, and change history
2. **During**: Trace impact across adjacent UOS repos before changing interfaces
3. **After**: Add tests, benchmarks, runbooks, or automation assets
4. **Escalate**: When action is irreversible, privacy-sensitive, or materially risky

## Document Types

- Postmortems: What went wrong and how to prevent recurrence
- Success stories: What worked and why it can be replicated
- Research notes: Unresolved questions and findings
- Improvement proposals: Ideas for future iterations
